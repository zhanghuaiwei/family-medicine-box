# 家庭药箱系统开发计划（个人版）

## 项目概述

开发一个个人药箱管理系统，包括Web管理端和微信小程序端，解决个人药品管理问题，实现药品使用监控、常用药品推荐和低库存报警功能。

## 技术栈

* **前端**：Vue 3 + Element Plus + Vite + PNPM + TypeScript + Pinia + Less + 响应式css + 动画效果css

* **微信小程序**：uni-app + Vue 3 + TypeScript

* **服务端**：JDK 1.8 + Maven + Spring Boot 微服务

* **数据库**：MySQL 8+

## 项目结构

### 1. 后端系统 (backend)

* **模块划分**：

  * `medicine-box-gateway`：API网关，统一请求入口

  * `medicine-box-biz`：业务逻辑模块，

    * `medicine-service`：药品管理服务，

      * 药品CRUD操作，

      * 药品分类管理，

      * 药品推荐算法。

    * `inventory-service`：库存管理服务，

      * 库存监控，

      * 低库存预警，

      * 库存变动记录。

    * `alert-service`：报警服务，

      * 短信报警，

      * 微信通知，

      * 邮件提醒。

    * `user-service`：用户管理服务，

      * 用户认证，

      * 权限控制。

  * `medicine-box-utils`：工具类模块，

    * 通用工具，

    * 配置管理，

    * 常量定义。

  * `medicine-box-common`：公共组件模块，

    * 数据模型，

    * 异常处理，

    * 响应结构。

* **核心功能**：

  * 药品管理（CRUD）

  * 库存监控与预警

  * 多渠道报警系统

  * 用户管理

  * 药品推荐系统

### 2. 前端系统 (frontend)

* **Web管理端**：

  * 响应式设计，适配PC和移动设备

  * 药品管理界面（增删改查）

  * 库存监控仪表盘（实时数据、趋势图表）

  * 报警设置与管理

  * 用户管理

  * 药品推荐界面（按人群、季节等）

* **微信小程序**：

  * 采用uni-app框架

  * 药品查询与搜索

  * 库存查看与管理

  * 用药记录与提醒

  * 低库存提醒

  * 微信登录与绑定

  * 消息推送功能

### 3. 数据库设计

* **主要表结构**：

  1. **users**：用户信息

     * id (VARCHAR(36), PK)：用户ID（UUID）

     * username (VARCHAR(50), NOT NULL)：用户名

     * password (VARCHAR(100), NOT NULL)：密码（加密存储）

     * phone (VARCHAR(20), NOT NULL, UNIQUE)：手机号码

     * email (VARCHAR(100), UNIQUE)：邮箱

     * nickname (VARCHAR(50))：昵称

     * avatar (VARCHAR(255))：头像URL

     * gender (TINYINT)：性别（0-未知，1-男，2-女）

     * birthdate (DATE)：出生日期

     * role (TINYINT, NOT NULL, DEFAULT 0)：角色（0-普通用户，1-管理员）

     * status (TINYINT, NOT NULL, DEFAULT 1)：状态（1-正常，0-禁用）

     * last\_login\_time (DATETIME)：最后登录时间

     * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

     * update\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP)：更新时间

     * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  2. **medicine\_categories**：药品分类

     * id (VARCHAR(36), PK)：分类ID（UUID）

     * category\_name (VARCHAR(50), NOT NULL)：分类名称

     * parent\_id (VARCHAR(36), FK)：父分类ID

     * description (VARCHAR(255))：分类描述

     * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

     * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  3. **medicines**：药品信息

     * id (VARCHAR(36), PK)：药品ID（UUID）

     * name (VARCHAR(100), NOT NULL)：药品名称

     * category\_id (VARCHAR(36), FK)：分类ID

     * brand (VARCHAR(100))：品牌

     * specification (VARCHAR(100), NOT NULL)：规格

     * dosage\_form (VARCHAR(50))：剂型（片剂、胶囊、颗粒等）

     * usage (TEXT)：用法用量

     * indications (TEXT)：适应症

     * contraindications (TEXT)：禁忌症

     * side\_effects (TEXT)：不良反应

     * storage\_condition (VARCHAR(255))：储存条件

     * expiry\_date\_days (INT)：保质期（天数）

     * manufacturer (VARCHAR(100))：生产厂家

     * approval\_number (VARCHAR(50))：批准文号

     * price (DECIMAL(10,2))：价格

     * barcode (VARCHAR(50), UNIQUE)：条形码

     * status (TINYINT, NOT NULL, DEFAULT 1)：状态（1-正常，0-停用）

     * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

     * update\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP)：更新时间

     * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  4. **inventory**：库存记录

     * id (VARCHAR(36), PK)：库存ID（UUID）

     * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

     * medicine\_id (VARCHAR(36), NOT NULL, FK)：药品ID

     * batch\_number (VARCHAR(50))：批次号

     * purchase\_date (DATE)：购买日期

     * expiry\_date (DATE, NOT NULL)：过期日期

     * current\_quantity (INT, NOT NULL)：当前数量

     * min\_stock (INT, NOT NULL, DEFAULT 1)：最低库存

     * unit (VARCHAR(20), NOT NULL)：单位（瓶、盒、片等）

     * location (VARCHAR(100))：存放位置

     * last\_update\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP)：最后更新时间

     * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  5. **inventory\_history**：库存变动记录

     * id (VARCHAR(36), PK)：记录ID（UUID）

     * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

     * inventory\_id (VARCHAR(36), NOT NULL, FK)：库存ID

     * medicine\_id (VARCHAR(36), NOT NULL, FK)：药品ID

     * change\_type (TINYINT, NOT NULL)：变动类型（1-增加，2-减少）

     * change\_quantity (INT, NOT NULL)：变动数量

     * before\_quantity (INT, NOT NULL)：变动前数量

     * after\_quantity (INT, NOT NULL)：变动后数量

     * reason (VARCHAR(255))：变动原因

     * change\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：变动时间

  6. **usage\_records**：用药记录

     * id (VARCHAR(36), PK)：记录ID（UUID）

     * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

     * medicine\_id (VARCHAR(36), NOT NULL, FK)：药品ID

     * usage\_quantity (INT, NOT NULL)：使用数量

     * usage\_time (DATETIME, NOT NULL)：使用时间

     * notes (VARCHAR(255))：备注

     * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

  7. **alerts**：报警记录

     * id (VARCHAR(36), PK)：报警ID（UUID）

     * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

     * medicine\_id (VARCHAR(36), FK)：药品ID

     * alert\_type (TINYINT, NOT NULL)：报警类型（1-低库存，2-即将过期，3-已过期）

     * alert\_level (TINYINT, NOT NULL)：报警级别（1-提醒，2-警告，3-严重）

     * message (VARCHAR(255), NOT NULL)：报警消息

     * status (TINYINT, NOT NULL, DEFAULT 0)：状态（0-未处理，1-已处理）

     * send\_time (DATETIME)：发送时间

     * resolve\_time (DATETIME)：解决时间

     * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

  8. **alert\_settings**：报警设置

  * id (VARCHAR(36), PK)：设置ID（UUID）

  * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

  * alert\_type (TINYINT, NOT NULL)：报警类型

  * alert\_level (TINYINT, NOT NULL)：报警级别

  * threshold (INT)：阈值（如低库存阈值）

  * enabled (TINYINT, NOT NULL, DEFAULT 1)：是否启用

  * notification\_methods (VARCHAR(50))：通知方式（短信、微信、邮件，逗号分隔）

  * update\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP)：更新时间

  * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  1. **medicine\_recommendations**：药品推荐

  * id (VARCHAR(36), PK)：推荐ID（UUID）

  * category (VARCHAR(50), NOT NULL)：推荐类别（按人群、季节等）

  * target\_group (VARCHAR(50))：目标人群（小孩、成人、老人、男人、女人）

  * season (VARCHAR(20))：季节（春、夏、秋、冬）

  * recommended\_medicines (TEXT)：推荐药品列表（JSON格式）

  * description (VARCHAR(255))：推荐说明

  * create\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：创建时间

  * update\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP)：更新时间

  * deleted (TINYINT, NOT NULL, DEFAULT 0)：逻辑删除（0-未删除，1-已删除）

  1. **notifications**：通知记录

  * id (VARCHAR(36), PK)：通知ID（UUID）

  * user\_id (VARCHAR(36), NOT NULL, FK)：用户ID

  * type (TINYINT, NOT NULL)：通知类型（1-系统通知，2-报警通知，3-用药提醒）

  * title (VARCHAR(100), NOT NULL)：通知标题

  * content (TEXT, NOT NULL)：通知内容

  * is\_read (TINYINT, NOT NULL, DEFAULT 0)：是否已读

  * send\_time (DATETIME, NOT NULL, DEFAULT CURRENT\_TIMESTAMP)：发送时间

  * read\_time (DATETIME)：阅读时间
